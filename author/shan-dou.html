Notes on Improving Python Code Efficiency (1)

Published: Sat 20 March 2021
By Shan Dou

In Good Practice.

tags: python engineering

An interesting learning on code efficiency today is on the performance difference between formal and literal definitions of common python data structures. In a nutshell, literal definition is more performant and should be preferred (while having readability in mind).

Formal vs. Literal Definition

dictionary list tuple
Formal definition dict() list() tuple()
Literal definition {} [] ()

Run Time Comparisons

In [1]: %timeit -r10 -n100 list()
86.9 ns ± 2.21 ns per loop (mean ± std. dev. of 10 runs, 100 loops each)

In [2]: %timeit -r10 -n100 []
26.3 ns ± 0.98 ns per loop (mean ± std. dev. of 10 runs, 100 loops each)

We can see from this example that the literal definition is about 3 times faster than its formal-definition counterpart.

%timeit vs. %%timeit

  • %timeit is a line magic that allows us to check runtime for a single line of code
  • Usage %timeit <line of code>
  • %%timeit is a cell magic that is for checking runtime for multiple lines of code
  • Usage %%timeit [Shift + Enter] <multiple lines of code>
